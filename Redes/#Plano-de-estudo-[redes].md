## COMO ESTUDAR

### LIVRO 1: Redes de Computadores Para Leigos
**Duração estimada:** 2 meses

**Como estudar este livro:**

1. **Leitura descontraída:** Leia como se fosse um romance técnico, sem pressão; marque frases que não entendeu para reler depois.

2. **Glossário próprio:** Crie um caderno de "dicionário de redes" anotando cada termo novo (switch, router, IP, DNS) com definição em suas palavras.

3. **Desenhos práticos:** Desenhe sua rede de casa identificando cada equipamento; depois desenhe redes maiores (escritório, escola).

4. **Experimentos mentais:** Para cada conceito, imagine aplicando em situações reais (ex: "como configuraria Wi-Fi de uma cafeteria?").

5. **Teste de domínio:** Consiga explicar para alguém leigo (ou para si mesmo) o que é IP, como funciona internet, diferença entre modem/roteador/switch.

---

### LIVRO 2: Redes de Computadores e a Internet
**Duração estimada:** 3-4 meses

**Como estudar este livro:**

1. **Leitura conectada:** Antes de cada capítulo, revise suas anotações do Livro 1 sobre o tema; leia conectando teoria nova com prática antiga.

2. **Mapas de camadas:** Crie um grande diagrama mostrando as 5 camadas da internet (aplicação → transporte → rede → enlace → física).

3. **Cartões de protocolos:** Para cada protocolo estudado, faça um cartão: nome, função, porta, exemplo real de uso, mensagens principais.

4. **Exercícios do livro:** Resolva TODOS os exercícios propostos; os problemas numéricos de cálculo são essenciais para dominar conceitos.

5. **Teste de domínio:** Consiga traçar no papel o caminho completo de uma requisição HTTP desde o navegador até o servidor web.

---

### LIVRO 3: Redes de Computadores (Tanenbaum)
**Duração estimada:** 4-5 meses

**Como estudar este livro:**

1. **Leitura técnica profunda:** Leia devagar, estude tabelas e diagramas com atenção; não pule seções técnicas mesmo que pareçam difíceis.

2. **Comparação de tecnologias:** Crie tabelas comparando tecnologias similares (Ethernet vs Wi-Fi, IPv4 vs IPv6, TCP vs UDP vs SCTP).

3. **Análise de RFCs:** Quando o livro citar uma RFC, anote o número e faça resumo de 1 página sobre ela (RFC 791-IP, RFC 793-TCP).

4. **Simulações complexas:** Desenhe redes empresariais completas simulando roteamento, falhas, balanceamento de carga, segurança em múltiplas camadas.

5. **Teste de domínio:** Consiga desenhar arquitetura completa de uma rede corporativa com justificativa técnica para cada decisão (topologia, protocolos, equipamentos).

---

### REVISÃO INTEGRADA FINAL

**Quando fazer:** Após concluir os 3 livros

**Como revisar tudo junto:**

1. **Mapa da jornada:** Crie um documento visual mostrando sua evolução: prática básica → teoria didática → domínio técnico avançado.

2. **Projeto mega:** Projete no papel uma infraestrutura completa: rede residencial + empresarial + data center + conexões WAN, com todos os protocolos, equipamentos, cálculos de endereçamento e medidas de segurança.

3. **Teste final integrado:** Pegue 15 perguntas difíceis (5 de cada livro) e responda sem consultar; depois revise gaps que ainda tem.

**Duração:** 3 semanas

**Resultado esperado:**  
Você dominará redes desde conceitos práticos básicos (montar rede de casa) até teoria avançada (projetar infraestrutura de empresa multinacional). Estará pronto para certificações (CCNA, Network+) ou para trabalhar com redes profissionalmente.
